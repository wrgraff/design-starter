# PWA

The app ships as a **Progressive Web App** ‚Äî installable, offline-aware, with a manifest and a service worker. Implementation uses [`@vite-pwa/sveltekit`](https://vite-pwa-org.netlify.app/frameworks/sveltekit).

## What's Included

- **Web App Manifest** ‚Äî `static/manifest.webmanifest` defines name, icons, theme color, display mode (standalone).
- **Service worker** ‚Äî generated by Workbox via `vite-plugin-pwa`. Strategy: `autoUpdate` (new versions install in the background, activate on next reload).
- **Pre-cached app shell** ‚Äî HTML, JS, CSS, fonts, icons cached on first visit.
- **Offline fallback** ‚Äî when a navigation cannot reach the network, the cached shell renders.
- **Installable** ‚Äî browsers offer the install prompt where supported (Chromium-based, Safari iOS via Share ‚Üí Add to Home Screen, etc.).

## Configuration

The single config lives in `vite.config.ts`, in the `SvelteKitPWA({ ... })` plugin invocation. Edit there:

- `manifest.name`, `manifest.short_name`, `manifest.description` ‚Äî set per project.
- `manifest.theme_color`, `manifest.background_color` ‚Äî usually `--color-primary` and `--color-background` from your tokens.
- `manifest.icons` ‚Äî paths to the icon files in `static/icons/`.
- `workbox.navigateFallbackDenylist` ‚Äî paths that should never serve the cached shell (e.g. API routes ‚Äî already excluded).

When `pnpm init-project` runs, it fills name/description/theme_color from your answers.

## Icons

Two purposes:

- **Any** ‚Äî used by browsers and OSes that draw the icon as-is (with possible rounding).
- **Maskable** ‚Äî used by Android/Chromium for adaptive icons; the OS applies a mask (square, circle, squircle). Maskable icons should have a generous safe zone (~10% padding on all sides).

Sizes shipped by default:

- `static/icons/icon-192.png` (192√ó192, any)
- `static/icons/icon-512.png` (512√ó512, any)
- `static/icons/icon-maskable-512.png` (512√ó512, maskable, with safe zone)

You can also add `icon-180.png` (Apple touch icon) referenced from `app.html`.

### Generating Icons

The fastest path: produce one square master at 1024√ó1024 (or higher) and let a generator slice the sizes. Recommended:

- [PWA Asset Generator](https://github.com/onderceylan/pwa-asset-generator) ‚Äî CLI, generates all required sizes including Apple splash screens.
- [maskable.app](https://maskable.app/) ‚Äî interactive tool for previewing and exporting maskable icons.

Drop the generated files into `static/icons/` and verify paths in `vite.config.ts` match.

## Theme Color

`theme_color` in the manifest controls the OS / browser chrome (status bar tint on Android, title bar on desktop installs). It should match the brand's primary color in light mode.

For dark mode chrome, browsers do not currently switch manifest `theme_color` automatically. To support dark theme chrome, add a `<meta>` tag to `app.html`:

```html
<meta name="theme-color" media="(prefers-color-scheme: light)" content="#ffffff" />
<meta name="theme-color" media="(prefers-color-scheme: dark)" content="#0a0a0a" />
```

These override the manifest value when the user's OS theme matches.

## Update Strategy

We use `autoUpdate`:

1. The user opens the app.
2. The service worker checks for a new version in the background.
3. If a new version is found, it is downloaded and activated on the next navigation/reload.

This is the lowest-friction strategy. If you want to _prompt_ the user before activating (e.g. for apps with in-flight state), switch to `prompt` and show a toast via the `usePwaUpdate` helper. Off by default ‚Äî most projects do not need it.

## Testing the PWA

### Manifest

```bash
pnpm build
pnpm preview
```

Then open <http://localhost:4173> in Chromium and check **DevTools ‚Üí Application ‚Üí Manifest**. The manifest should parse without warnings, all icons load, and "Installability" should show **App is installable**.

### Service worker

DevTools ‚Üí Application ‚Üí Service workers. The worker should be `activated and running`. Click **Update** to force a refresh.

### Offline mode

DevTools ‚Üí Network ‚Üí throttle to **Offline**. Reload the page ‚Äî the cached shell should render. Try navigating to a route you have visited before; it should work. Routes you have never visited may show the offline fallback or fail (which is expected ‚Äî we don't pre-cache everything).

### Install

Chromium: address-bar install icon (varies by URL). Once installed, the app opens in a standalone window without browser chrome.

iOS Safari: Share ‚Üí Add to Home Screen. Note: iOS PWAs have limitations (no push, limited storage, periodic eviction).

### Lighthouse audit

DevTools ‚Üí Lighthouse ‚Üí Progressive Web App. Aim for green across the board. Common warnings:

- `apple-touch-icon` missing ‚Üí add `<link rel="apple-touch-icon" href="/icons/icon-180.png">` in `app.html`.
- `maskable` icon missing ‚Üí check `icon-maskable-512.png` is referenced with `purpose: 'maskable'` in the manifest.

## Limitations and Gotchas

- **Service workers do not run in `pnpm dev` by default.** `devOptions.enabled: false` in `vite.config.ts`. Use `pnpm build && pnpm preview` to test the SW.
- **iOS PWAs have storage quotas and are evicted after ~7 days of inactivity.** Plan for re-fetch on cold start.
- **Push notifications** require a separate setup (web-push) and user permission. Not included by default.
- **Background sync / periodic sync** is Chromium-only and requires the user to install the app. Not included by default.
- The first request to a hashed asset (`/_app/immutable/...`) is always network ‚Äî only subsequent requests use the cache. So a fresh install must reach the network for the app to work.

## Disabling the PWA

If a project doesn't need PWA features at all, remove the plugin call from `vite.config.ts`:

```diff
- import { SvelteKitPWA } from '@vite-pwa/sveltekit';

  plugins: [
    tailwindcss(),
    sveltekit(),
-   SvelteKitPWA({ ... })
  ]
```

And remove `@vite-pwa/sveltekit` from `package.json`. The app still works ‚Äî it just won't be installable and won't cache for offline.
